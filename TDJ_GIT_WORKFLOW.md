# Tie Dye Jedi GitHub Workflow

This document explains how Tie Dye Jedi uses GitHub.

## Branches

### master

Stable robot code.

Code in master should be tested and suitable for demonstrations, scrimmages, or competition.

### development

Active team development branch.

Most coding work will eventually be merged here.

### Feature Branches

Students should create their own branches from development.

Examples:

- motor-test
- servo-test
- autonomous-practice
- your-name-feature

## Standard Workflow

### Step 1

Pull the latest changes from development.

### Step 2

Create a new branch from development.

### Step 3

Make your changes.

### Step 4

Commit your changes.

Commit messages should briefly describe what was changed.

Examples:

- Added motor test op mode
- Fixed drivetrain controls
- Added telemetry output

### Step 5

Push your branch to GitHub.

### Step 6

Create a Pull Request into development.

### Step 7

Have another team member, coding lead, or mentor review the changes.

### Step 8

Merge into development after approval.

## Philosophy

- Make small changes.
- Test frequently.
- Commit often.
- Ask questions.
- Help each other.

Remember:

A working robot is more valuable than clever code.
